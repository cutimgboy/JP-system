# 架构设计方案

## 一、概述

本文档基于现有项目架构（React Router v7 + SSR），设计并规划以下三个核心功能的架构方案：

1. **一键切换多语言** - 国际化（i18n）系统
2. **统一的请求管理** - HTTP 请求层架构
3. **合理的路由配置** - 路由组织与配置方案

---

## 二、多语言架构设计

### 2.1 技术选型

**推荐方案**: `react-i18next` + `i18next`

**理由**:
- 成熟稳定，社区活跃
- 支持 SSR（与 React Router v7 兼容）
- 支持懒加载语言包
- 类型安全（TypeScript 支持）
- 支持命名空间（namespace）管理

### 2.2 目录结构

```
app/
├── i18n/                    # 国际化目录
│   ├── config.ts           # i18n 配置文件
│   ├── resources/          # 语言资源文件
│   │   ├── zh-CN/          # 简体中文
│   │   │   ├── common.json # 通用翻译
│   │   │   ├── home.json   # 首页翻译
│   │   │   └── ...
│   │   ├── en-US/          # 英文
│   │   │   ├── common.json
│   │   │   ├── home.json
│   │   │   └── ...
│   │   └── ja-JP/          # 日文（示例）
│   │       └── ...
│   └── locales.ts          # 语言列表定义
├── hooks/                   # 自定义 Hooks
│   └── useLanguage.ts      # 语言切换 Hook
└── components/             # 公共组件
    └── LanguageSwitcher/   # 语言切换组件
        └── index.tsx
```

### 2.3 架构设计

#### 2.3.1 i18n 初始化配置

**文件**: `app/i18n/config.ts`

**核心功能**:
- 初始化 i18next 实例
- 配置语言检测（浏览器语言、URL 参数、localStorage）
- 配置语言资源加载方式
- 支持 SSR 环境

**关键配置项**:
```typescript
{
  // 默认语言
  fallbackLng: 'zh-CN',
  // 支持的语言列表
  supportedLngs: ['zh-CN', 'en-US'],
  // 命名空间
  defaultNS: 'common',
  // 资源加载方式（按需加载）
  load: 'languageOnly',
  // SSR 支持
  react: { useSuspense: false }
}
```

#### 2.3.2 语言资源管理

**文件结构**:
- `resources/zh-CN/common.json` - 通用翻译（按钮、表单、错误信息等）
- `resources/zh-CN/home.json` - 首页相关翻译
- `resources/zh-CN/routes/*.json` - 各路由页面翻译

**资源文件示例**:
```json
// resources/zh-CN/common.json
{
  "button": {
    "submit": "提交",
    "cancel": "取消",
    "confirm": "确认"
  },
  "error": {
    "404": "页面未找到",
    "500": "服务器错误"
  }
}
```

#### 2.3.3 语言切换机制

**实现方式**:
1. **Context + Hook**: 使用 React Context 管理当前语言状态
2. **URL 参数**: 通过路由参数 `/zh-CN/home` 或 `/en-US/home` 切换
3. **localStorage**: 持久化用户语言偏好
4. **一键切换组件**: 提供语言选择器 UI

**语言切换流程**:
```
用户点击语言切换
  │
  ▼
更新 Context 状态
  │
  ├─> 更新 i18next 语言
  ├─> 更新 localStorage
  ├─> 更新 URL（可选）
  └─> 触发组件重新渲染
```

#### 2.3.4 SSR 支持

**关键点**:
- 服务端初始化 i18n 时，从请求头 `Accept-Language` 或 URL 参数获取语言
- 客户端水合（hydration）时，与服务端语言保持一致
- 语言资源在服务端预加载，避免闪烁

### 2.4 使用示例

**在组件中使用**:
```typescript
import { useTranslation } from 'react-i18next';

function HomePage() {
  const { t } = useTranslation('home');
  
  return (
    <div>
      <h1>{t('title')}</h1>
      <p>{t('description')}</p>
    </div>
  );
}
```

**语言切换组件**:
```typescript
function LanguageSwitcher() {
  const { i18n } = useTranslation();
  
  const changeLanguage = (lng: string) => {
    i18n.changeLanguage(lng);
    // 可选：更新 URL
    // navigate(`/${lng}${pathname}`);
  };
  
  return (
    <select onChange={(e) => changeLanguage(e.target.value)}>
      <option value="zh-CN">中文</option>
      <option value="en-US">English</option>
    </select>
  );
}
```

## 三、统一请求管理架构设计

### 3.1 技术选型

**推荐方案**: `axios` + 自定义封装

**理由**:
- 功能完善，支持拦截器
- 支持请求/响应拦截
- 支持取消请求
- 支持 TypeScript
- 兼容 SSR 环境

### 3.2 目录结构

```
app/
├── lib/                     # 工具库目录
│   └── api/                 # API 请求层
│       ├── client.ts        # HTTP 客户端实例
│       ├── interceptors.ts # 拦截器配置
│       ├── types.ts         # 类型定义
│       └── endpoints/       # API 端点定义
│           ├── user.ts      # 用户相关 API
│           ├── product.ts   # 产品相关 API
│           └── ...
├── hooks/                   # 自定义 Hooks
│   └── useApi.ts           # API 请求 Hook
└── utils/                    # 工具函数
    └── errorHandler.ts     # 错误处理工具
```

### 3.3 架构设计

#### 3.3.1 HTTP 客户端封装

**文件**: `app/lib/api/client.ts`

**核心功能**:
- 创建 axios 实例
- 配置基础 URL（从环境变量读取）
- 配置默认请求头
- 配置超时时间
- 配置请求/响应拦截器

**关键配置**:
```typescript
const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});
```

#### 3.3.2 拦截器设计

**文件**: `app/lib/api/interceptors.ts`

**请求拦截器**:
- 添加认证 Token（从 localStorage 或 cookie）
- 添加语言标识（从 i18n context）
- 添加请求 ID（用于日志追踪）
- 请求参数序列化

**响应拦截器**:
- 统一错误处理（401、403、500 等）
- 响应数据格式化
- Token 过期自动刷新（可选）
- 请求日志记录

**错误处理流程**:
```
API 请求失败
  │
  ├─> 网络错误 → 提示"网络连接失败"
  ├─> 401 未授权 → 清除 Token，跳转登录页
  ├─> 403 无权限 → 提示"无权限访问"
  ├─> 404 未找到 → 提示"资源不存在"
  ├─> 500 服务器错误 → 提示"服务器错误，请稍后重试"
  └─> 其他错误 → 显示错误消息
```

#### 3.3.3 API 端点管理

**文件**: `app/lib/api/endpoints/user.ts`

**组织方式**:
- 按业务模块划分文件
- 每个端点定义请求方法、路径、参数类型、响应类型
- 使用 TypeScript 确保类型安全

**示例**:
```typescript
// endpoints/user.ts
export const userApi = {
  // 获取用户信息
  getUserInfo: (userId: string): Promise<UserInfo> => {
    return apiClient.get(`/users/${userId}`);
  },
  
  // 更新用户信息
  updateUserInfo: (userId: string, data: UpdateUserDto): Promise<UserInfo> => {
    return apiClient.put(`/users/${userId}`, data);
  },
  
  // 登录
  login: (credentials: LoginDto): Promise<LoginResponse> => {
    return apiClient.post('/auth/login', credentials);
  },
};
```

#### 3.3.4 React Hook 封装

**文件**: `app/hooks/useApi.ts`

**功能**:
- 封装请求状态（loading、error、data）
- 支持自动请求和手动触发
- 支持请求取消
- 支持缓存（可选）

**使用示例**:
```typescript
function UserProfile({ userId }: { userId: string }) {
  const { data, loading, error, refetch } = useApi(
    () => userApi.getUserInfo(userId),
    [userId]
  );
  
  if (loading) return <Loading />;
  if (error) return <Error message={error.message} />;
  
  return <div>{data?.name}</div>;
}
```

#### 3.3.5 SSR 支持

**关键点**:
- 服务端请求时，从请求头获取 Cookie（包含 Token）
- 服务端请求时，使用服务端 API 地址
- 客户端请求时，使用相对路径或客户端 API 地址
- 避免在服务端请求时暴露敏感信息

### 3.4 类型定义

**文件**: `app/lib/api/types.ts`

**核心类型**:
```typescript
// API 响应包装类型
export interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
}

// 分页响应类型
export interface PaginatedResponse<T> {
  items: T[];
  total: number;
  page: number;
  pageSize: number;
}

// 错误响应类型
export interface ApiError {
  code: number;
  message: string;
  details?: any;
}
```

## 四、路由配置架构设计

### 4.1 路由组织原则

1. **按功能模块划分**: 每个业务模块独立路由文件
2. **嵌套路由**: 利用 React Router v7 的嵌套路由特性
3. **路由守卫**: 实现权限控制和认证检查
4. **懒加载**: 路由组件按需加载，提升性能
5. **类型安全**: 利用 TypeScript 和 React Router 类型系统

### 4.2 目录结构

```
app/
├── routes.ts                # 路由配置文件（主入口）
├── routes/                  # 路由组件目录
│   ├── _index.tsx          # 首页路由
│   ├── _layout.tsx         # 布局路由（可选）
│   ├── auth/               # 认证模块路由
│   │   ├── login.tsx       # 登录页
│   │   └── register.tsx    # 注册页
│   ├── dashboard/          # 仪表盘模块路由
│   │   ├── _layout.tsx     # 仪表盘布局
│   │   ├── index.tsx       # 仪表盘首页
│   │   └── settings.tsx    # 设置页
│   ├── products/           # 产品模块路由
│   │   ├── _layout.tsx     # 产品布局
│   │   ├── index.tsx       # 产品列表
│   │   └── $id.tsx         # 产品详情（动态路由）
│   └── ...
├── layouts/                # 布局组件目录
│   ├── MainLayout.tsx      # 主布局
│   ├── AuthLayout.tsx      # 认证布局
│   └── DashboardLayout.tsx # 仪表盘布局
└── guards/                 # 路由守卫
    ├── authGuard.ts        # 认证守卫
    └── roleGuard.ts        # 角色守卫
```

### 4.3 路由配置方案

#### 4.3.1 主路由配置

**文件**: `app/routes.ts`

**配置方式**:
- 使用 React Router v7 的路由配置 API
- 支持嵌套路由
- 支持路由元数据（meta）
- 支持路由加载器（loader）和动作（action）

**示例配置**:
```typescript
import { type RouteConfig, index, route, layout } from "@react-router/dev/routes";

export default [
  // 首页
  index("routes/_index.tsx"),
  
  // 认证模块
  route("auth/login", "routes/auth/login.tsx"),
  route("auth/register", "routes/auth/register.tsx"),
  
  // 仪表盘模块（嵌套路由）
  layout("routes/dashboard/_layout.tsx", [
    index("routes/dashboard/index.tsx"),
    route("settings", "routes/dashboard/settings.tsx"),
  ]),
  
  // 产品模块（嵌套路由）
  layout("routes/products/_layout.tsx", [
    index("routes/products/index.tsx"),
    route(":id", "routes/products/$id.tsx"),
  ]),
  
  // 404 页面
  route("*", "routes/404.tsx"),
] satisfies RouteConfig;
```

#### 4.3.2 路由元数据

**用途**:
- 定义页面标题、描述（SEO）
- 定义权限要求
- 定义是否需要认证
- 定义布局类型

**实现方式**:
在路由组件中导出 `meta` 函数：
```typescript
// routes/dashboard/index.tsx
export const meta: Route.MetaFunction = ({ data }) => {
  return [
    { title: "仪表盘" },
    { name: "description", content: "用户仪表盘页面" },
  ];
};
```

#### 4.3.3 路由守卫

**文件**: `app/guards/authGuard.ts`

**实现方式**:
- 在路由的 `loader` 函数中检查认证状态
- 未认证则重定向到登录页
- 已认证则返回用户信息

**示例**:
```typescript
// routes/dashboard/_layout.tsx
export const loader: Route.LoaderFunction = async ({ request }) => {
  const user = await getCurrentUser(request);
  
  if (!user) {
    throw redirect("/auth/login");
  }
  
  return { user };
};
```

#### 4.3.4 路由懒加载

**实现方式**:
- 使用 React Router v7 的 `lazy` 函数
- 或使用动态 `import()` 语法

**示例**:
```typescript
// routes.ts
import { lazy } from "react-router";

const Dashboard = lazy(() => import("./routes/dashboard/index.tsx"));

export default [
  route("dashboard", Dashboard),
] satisfies RouteConfig;
```

#### 4.3.5 多语言路由支持

**方案**:
- 在路由路径中包含语言参数：`/:lang/dashboard`
- 或在路由配置中为每种语言创建独立路由
- 使用路由参数：`/dashboard?lang=zh-CN`

**推荐方案**: URL 参数方式（更灵活，不影响路由结构）

### 4.4 路由最佳实践

1. **路由命名规范**:
   - 使用 kebab-case: `user-profile.tsx`
   - 动态路由使用 `$` 前缀: `$id.tsx`
   - 布局路由使用 `_layout.tsx`

2. **路由分组**:
   - 按业务模块分组
   - 相关路由放在同一目录

3. **路由权限**:
   - 在 `loader` 中检查权限
   - 使用路由守卫统一处理

4. **SEO 优化**:
   - 每个路由定义 `meta` 函数
   - 支持服务端渲染（SSR）

## 五、集成方案

### 5.1 三个模块的协作关系

```
┌─────────────────────────────────────────┐
│            React Router v7              │
│  ┌───────────────────────────────────┐  │
│  │        路由配置 (routes.ts)        │  │
│  │  ┌─────────────────────────────┐  │  │
│  │  │  路由守卫 (authGuard)        │  │  │
│  │  └─────────────────────────────┘  │  │
│  └───────────────────────────────────┘  │
└──────────────┬──────────────────────────┘
               │
    ┌──────────┴──────────┐
    │                     │
    ▼                     ▼
┌──────────┐      ┌──────────────┐
│  i18n    │      │  API Client  │
│ (多语言)  │      │ (请求管理)   │
└──────────┘      └──────────────┘
    │                     │
    └──────────┬──────────┘
               │
               ▼
        ┌──────────────┐
        │   组件层      │
        │  (业务逻辑)   │
        └──────────────┘
```

### 5.2 初始化顺序

1. **应用启动** → 初始化 i18n
2. **路由匹配** → 执行路由守卫
3. **组件加载** → 使用 i18n 和 API Client

### 5.3 数据流

```
用户请求
  │
  ├─> 路由匹配 (routes.ts)
  │   │
  │   ├─> 路由守卫检查 (authGuard)
  │   │   └─> API 请求验证 (apiClient)
  │   │
  │   └─> 加载路由组件
  │       │
  │       ├─> 使用 i18n 翻译文本
  │       │
  │       └─> 使用 API Client 请求数据
  │
  └─> 渲染页面
```
