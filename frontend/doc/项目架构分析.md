# 项目架构分析文档

## 一、整体架构

### 1.1 技术栈

本项目是一个基于 **React Router v7** 的全栈 Web 应用，采用以下技术栈：

- **框架**: React Router v7.12.0 (支持 SSR)
- **构建工具**: Vite 7.1.7
- **语言**: TypeScript 5.9.2
- **样式**: Tailwind CSS 4.1.13
- **运行时**: Node.js 20

### 1.2 架构模式

```
┌─────────────────────────────────────────────────┐
│                  客户端浏览器                    │
└──────────────────┬──────────────────────────────┘
                   │ HTTP 请求
┌──────────────────▼──────────────────────────────┐
│          React Router Server (SSR)              │
│  ┌──────────────────────────────────────────┐   │
│  │  @react-router/node （服务端渲染引擎）     │   │
│  └──────────────────────────────────────────┘   │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              构建产物 (build/)                   │
│  ┌──────────────┐      ┌──────────────┐         │
│  │  client/     │      │  server/     │         │
│  │  (静态资源)  │      │  (服务端代码)  │         │
│  └──────────────┘      └──────────────┘         │
└─────────────────────────────────────────────────┘
```

## 二、项目入口及打包逻辑

### 2.1 开发入口

**开发命令**: `npm run dev`

执行流程：
```
npm run dev
  └─> react-router dev
      └─> 启动 Vite 开发服务器
          └─> 监听 app/ 目录变化
              └─> 热模块替换 (HMR)
```

**开发服务器地址**: `http://localhost:5173`

### 2.2 生产构建入口

**构建命令**: `npm run build`

执行流程：
```
npm run build
  └─> react-router build
      └─> 调用 Vite 进行构建
          ├─> 编译 TypeScript
          ├─> 处理 React 组件
          ├─> 打包静态资源
          └─> 生成构建产物
              ├─> build/client/  (客户端代码)
              └─> build/server/  (服务端代码)
```

### 2.3 生产运行入口

**运行命令**: `npm run start`

执行流程：
```
npm run start
  └─> react-router-serve ./build/server/index.js
      └─> 启动 Node.js 服务器
          └─> 提供 SSR 服务
```

## 三、文件详细说明

### 3.1 配置文件

#### `package.json`
- **作用**: 项目依赖管理和脚本定义
- **关键脚本**:
  - `dev`: 启动开发服务器
  - `build`: 构建生产版本
  - `start`: 启动生产服务器
  - `typecheck`: 类型检查
- **核心依赖**:
  - `react-router`: 路由框架
  - `@react-router/node`: 服务端渲染支持
  - `@react-router/serve`: 生产服务器

#### `vite.config.ts`
- **作用**: Vite 构建工具配置
- **插件配置**:
  - `tailwindcss()`: Tailwind CSS 支持
  - `reactRouter()`: React Router 集成
  - `tsconfigPaths()`: TypeScript 路径别名支持

#### `react-router.config.ts`
- **作用**: React Router 框架配置
- **关键配置**:
  - `ssr: true`: 启用服务端渲染

#### `tsconfig.json`
- **作用**: TypeScript 编译器配置
- **关键配置**:
  - `paths`: 路径别名 `~/*` 映射到 `./app/*`
  - `jsx: "react-jsx"`: React JSX 编译模式
  - `target: "ES2022"`: 编译目标版本

### 3.2 应用核心文件

#### `app/root.tsx`
- **作用**: 应用根组件，定义全局布局和错误处理
- **导出函数**:
  - `Layout`: HTML 文档结构（`<html>`, `<head>`, `<body>`）
  - `App`: 主应用组件，渲染 `<Outlet />` 显示子路由
  - `ErrorBoundary`: 错误边界组件，捕获路由错误
  - `links`: 定义外部资源链接（如 Google Fonts）

#### `app/routes.ts`
- **作用**: 路由配置文件，定义应用的所有路由
- **当前路由**:
  - `index("routes/home.tsx")`: 首页路由，映射到 `/`

#### `app/routes/home.tsx`
- **作用**: 首页路由组件
- **功能**:
  - 定义页面元数据（`meta` 函数）
  - 渲染 `Welcome` 组件

#### `app/welcome/welcome.tsx`
- **作用**: 欢迎页组件
- **功能**:
  - 显示 React Router Logo（支持深色/浅色模式）
  - 提供资源链接导航（文档、Discord 等）

#### `app/app.css`
- **作用**: 全局样式文件
- **内容**:
  - 导入 Tailwind CSS
  - 定义主题字体（Inter）
  - 配置深色模式支持

### 3.3 静态资源

#### `public/favicon.ico`
- **作用**: 网站图标

#### `app/welcome/logo-*.svg`
- **作用**: React Router Logo 图片（深色/浅色模式）

### 3.4 部署相关

#### `Dockerfile`
- **作用**: Docker 容器化构建配置
- **构建阶段**:
  1. `development-dependencies-env`: 安装开发依赖
  2. `production-dependencies-env`: 安装生产依赖
  3. `build-env`: 执行构建
  4. 最终镜像: 包含构建产物和运行依赖

## 四、数据流和请求处理

### 4.1 SSR 请求流程

```
用户请求
  │
  ▼
React Router Server
  │
  ├─> 匹配路由 (routes.ts)
  │
  ├─> 加载路由组件 (routes/home.tsx)
  │
  ├─> 执行服务端渲染
  │
  ├─> 生成 HTML
  │
  └─> 返回完整 HTML + 客户端脚本
```

### 4.2 客户端路由流程

```
用户交互 (点击链接/导航)
  │
  ▼
React Router Client
  │
  ├─> 匹配路由
  │
  ├─> 加载组件代码 (按需)
  │
  ├─> 更新 DOM
  │
  └─> 无需刷新页面
```

## 六、扩展建议

1. **添加新路由**: 在 `app/routes/` 目录下创建新文件，并在 `app/routes.ts` 中注册
2. **添加布局**: 在 `app/root.tsx` 的 `Layout` 组件中修改
3. **添加全局样式**: 在 `app/app.css` 中扩展
4. **环境变量**: 使用 `import.meta.env` 访问环境变量

## 七、交易页 K 线实现记录

记录日期：2026-05-23

### 7.1 设计稿实现方式

设计稿文件 `JPM 新版本UI/src/app/components/TradePage.tsx` 的 K 线区域使用 `recharts` 的 `AreaChart`/`Area` 渲染 SVG 折线和渐变面积，旗帜、最新价格点、倒计时、开仓价格线、看涨/看跌图标和收益浮窗使用 DOM 绝对定位叠加。

设计稿的核心动效不是传统蜡烛图，而是折线行情：

- 总槽位 `MAX_POINTS = 70`。
- 最新点先从左向右延伸。
- 到达 `FIXED_INDEX = 42` 后，最新点固定在中右位置，旧数据向左滚动。
- 上涨使用红色，下跌使用绿色。
- 最新点有呼吸动画，顶部有旗帜和竖向虚线。
- 下单后显示开仓价格水平线，以及看涨/看跌方向图标。

### 7.2 当前业务端实现方式

当前业务端 `frontend/src/pages/trading/components/KLineChart.tsx` 保留 canvas 实现，并按设计稿语义复刻上述动效。保留 canvas 的原因：

- 实时行情会持续接收 SSE tick，canvas 比 SVG + 大量 DOM overlay 更适合高频重绘。
- 最新点呼吸、趋势线滚动、开仓标记、倒计时和收益浮窗都可以在单画布内完成，减少 React 协调和布局计算压力。
- 画布只在尺寸或 DPR 变化时重建像素缓冲区，普通动画帧只清屏重绘。
- `requestAnimationFrame` 绘制循环读取 `renderStateRef` 中的最新 props，不再每帧 `setState` 触发 React 组件树重渲染。

### 7.3 与后端行情链路的配合

交易页 `TradingChart` 先请求 `GET /api/quote/kline/:code?interval=1s&limit=300` 加载快照，再接 `EventSource /api/quote/stream/:code`。SSE tick 在前端 200ms 合并后更新图表数据，避免每个 tick 都触发 React 更新。K 线组件内部再用 canvas RAF 做视觉补间，形成“数据低频更新 + 画布高频补间”的结构。

### 7.4 后续约束

1. 交易引导和图表 overlay 不应引入模拟手机状态栏。
2. 看涨保持红色，看跌保持绿色，和设计稿一致。
3. K 线最新点应固定在第 42 个槽位附近，不能回退为全量数据按时间铺满。
4. 如需增加交互能力，优先只在 canvas 内维护轻量状态，避免恢复每帧 React state。
