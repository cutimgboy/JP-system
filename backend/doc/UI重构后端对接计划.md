# UI 重构后端对接计划

更新时间：2026-05-23

## 背景

本次 UI 重构主要发生在 `frontend/`，但新版页面涉及交易、行情、账户、订单、资金、银行卡和用户资料等后端能力。后端已有 NestJS API，需要在前端迁移时优先复用，避免把新版 UI 的 mock 数据当成真实业务。

## 后端模块关注点

| 业务 | 相关后端模块/文件 | 前端用途 |
| --- | --- | --- |
| 产品与行情 | `backend/src/cfd/*`、`backend/src/quote/*` | 市场页、交易页产品切换、价格、涨跌幅、概况、交易时间。 |
| 交易订单 | `backend/src/user/services/trade-order.service.ts`、`/trade/*` | 下单、恢复进行中订单、持仓、历史订单、订单详情。 |
| 账户余额 | `backend/src/user/services/account.service.ts`、`/account/*` | 顶部账户选择、交易页余额、持仓统计、资金页余额。 |
| 银行卡 | `backend/src/bank-card/*` | 入金银行卡、提现银行卡、我的银行、新增银行卡。 |
| 系统收款卡 | `backend/src/system-bank-card/*` | 存入资金流程。 |
| 入金 | `backend/src/deposit/*` | 存款、上传凭证、入金详情、资金记录。 |
| 奖励/活动 | `backend/src/reward/*`、`backend/src/community/*` | 我的页权益、持仓奖励、排行榜/社区。 |
| 用户资料 | `backend/src/user/*`、`/user/info` | 个人资料、修改密码、登录后状态。 |
| 消息 | `backend/src/message/*` | 我的页消息中心。 |

## 后端可能需要补齐

- 提现申请接口：提交银行卡、金额、账户类型和身份认证信息。
- 提现记录列表接口：用于资金记录统一展示。
- 用户资料更新接口：修改昵称、头像、邮箱等。
- 产品详情接口字段补齐：交易时间、简介、概况、历史高低、市值等。如果后端暂不提供，前端可用产品基础字段 + 本地默认描述。
- 真实凭证/证件上传接口：当前入金页面把浏览器临时对象 URL 作为凭证地址，不适合刷新后查看或后台审核长期保存。

## 已确认接口与缺口

| 新增功能 | 对接结论 |
| --- | --- |
| 交易页重构 | 已有产品、行情、余额、下单、订单恢复 API，可直接保留当前业务逻辑。 |
| 订单详情和分享 | 订单详情 API 已有；分享可先由前端生成内容，不需要新增后端。 |
| 个人资料 | `GET /user/info`、`PUT /user/info`、密码接口已存在；部分展示字段可能为空。 |
| 提取资金 | 缺正式提现申请、审核、记录、详情接口；`POST /account/withdraw` 仅测试扣款，不适合作正式流程。 |
| 资金记录 | 入金和交易记录已有；提现记录和统一流水缺口仍在。 |
| 我的银行 | 用户银行卡 CRUD 已有。 |
| 安装 APP | 静态前端能力；如要后台配置下载链接，需要新增配置接口。 |
| 切换语言 | 当前可本地 i18n；跨设备保存可扩展 `PUT /user/info` 的语言偏好字段。 |

## 前端迁移原则

- 已有真实 API 的功能必须继续使用真实 API。
- 新版 UI 中的 mock 数组只作为视觉结构参考。
- 后端缺口先在页面中保留完整交互、校验和成功态，并在代码中收敛到单独函数，后续接 API 时替换成本较低。
- 订单、账户、资金相关页面必须保留错误处理、加载态和空态。

## 验收建议

- 登录后能进入 `/market` 并加载产品行情。
- demo/real 账户切换后余额、持仓和交易页同步变化。
- `/trading` 能选择标的、设置时间/金额、完成看涨/看跌下单。
- 有进行中订单时返回交易页能恢复倒计时。
- `/positions` 能展示交易中和历史订单。
- `/positions/order/:orderId` 能展示真实订单详情并触发分享弹层。
- `/my-bank` 能显示银行卡列表，`/my-bank/add` 能新增银行卡。
- `/fund-records` 能展示入金和交易资金记录。
- 提现流程页面完整，接口缺口清晰可替换。
